## 1. 금융주 식별 방식 (Phase 1)

### 사용 가능한 데이터 소스
- KIS API `bstp_kor_isnm` 응답이 `financial_metrics.sector`에 이미 주입됨.
- 분포 (2025 annual, n=233):
  - **'보험' 7건** ✓ 자동 분기 가능 (삼성화재/현대해상/DB손보/서울보증/한화생명/삼성생명/우선주1)
  - **'증권' 6건** ✓ 자동 분기 가능 (한화투자/미래에셋/삼성/키움/NH/우선주1)
  - **'금융' 32건** ⚠ 혼합 — 은행지주 11건 + **일반 지주 21건**(두산/CJ/LG/롯데지주/GS/HD현대/SK스퀘어 등)

### 권장 방식
1. `sector == '보험'`: 자동 (응답 내 라벨 패턴으로 다시 분기)
2. `sector == '증권'`: 자동 (영업수익 우선, 없으면 합산)
3. `sector == '금융'` + **종목코드 화이트리스트**: 은행/금융지주만 합산 룰 적용

**BANK_CODES (11)**: 신한지주, KB금융, 하나금융, 우리금융, 기업은행, 케이뱅크, 메리츠금융, iM금융, JB금융, 한국금융지주, BNK금융지주

DART 캐시 raw에는 sector 정보 없음. KRX 별도 분류 필요한 경우 신규 데이터 소스이지만 현재는 KIS sector + 화이트리스트로 충분.

## 3. 시뮬레이션 결과 (Phase 3)

### 변경/회복 종목
- **회복**: 11건 (BANK 0 → 정확값) + 1건 NH투자증권 + 1건 삼성생명
- **정정**: 6건 보험사 (단일 라인 → 합산)
- **변경 없음**: 한화투자/미래에셋/키움 (영업수익 정상 매칭, 회귀 0)
- **합계**: 19종목 회복/정정

### 비-금융 회귀 검증
- 상위 매출 30 표본 (전기·전자, 화학, 운송장비 등) 시뮬: **회귀 0건** ✓
- sector='금융' 일반 지주 21건 시뮬: 화이트리스트 외 코드는 sentinel `-1` 반환으로 변경 없음 → **회귀 0건** ✓

### 점수 영향 (추정)
- 보험사: 매출 4~5배 증가 → PSR 점수 호전, EV/EBITDA 점수 가능
- 은행지주: 0 → 수십조 → 매출 0이 만든 silent fail 점수 정정. 다음 사이클 신호 변동 가능.
- 한화투자증권 같은 정상 종목: **점수 변화 0** (sentinel 회귀 방지)

## 4. consecutive_*_years 보강 (Phase 4)

### 캐시 커버리지
| 연도 | 종목 수 | 비율 |
|---|---:|---:|
| 2025 | 223 | 100% |
| 2024 | 218 | 97.7% |
| **2023** | **35** | **15.7%** |

### 페널티 임계값 (settings.py TOTAL_SCORE_PENALTIES)
- `consecutive_revenue_decline_years >= 3` → -5점 (3년 캐시 필요)
- `prev_net_income > 0 AND net_income < 0` → -8점 (1년 비교, **캐시 1년만으로 가능**)
- `consecutive_loss_years >= 3` → -15점 (3년 캐시 필요)

### 효과 추정 (표본 48건 중)
- 흑→적자 전환 (1년 비교): 5건 후보 (직전 흑자 여부 추가 확인 필요)
- 3년 연속 적자 (3년 캐시 보유 35건 중): 1건만
- 3년 연속 매출 감소: 캐시 부족으로 정확 판정 거의 불가

### 결론
- **1년 비교는 효과 있으나 5건 수준** — 별도 PR 우선순위 낮음
- **3년 연속 페널티는 캐시 부재로 백필 불가** — 2023 annual 수집 누적 후 재평가
- 4-27 정상 분석 사이클에서 dart_api가 prev_df 호출 시 1년 비교는 자연 갱신 가능

## 6. 시뮬 검증 코드 (참고)

```python
# 핵심 분기 로직
def proposed_revenue(df, sector, code):
    if sector == '보험':
        a = exact('보험수익') + exact('투자영업수익')
        if a > 0: return a
        # IFRS17 (삼성생명)
        return exact('보험서비스수익') + exact('이자수익') + exact('수수료수익')
    if sector == '증권':
        op = exact('영업수익')
        if op > 0: return op
        return exact('수수료수익') + exact('이자수익')
    if sector == '금융' and code in BANK_CODES:
        return exact('이자수익') + exact('수수료수익') + exact('보험수익')
    return None  # 변경 없음 (sentinel)
```

회귀 테스트 시 sentinel 반환 시 기존 매출 보존 → 일반 종목 안전.
