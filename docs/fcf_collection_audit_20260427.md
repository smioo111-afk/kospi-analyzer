## 3. 코드 로직 분석

### FCF 계산 (collectors/dart_api.py:395-408)

```python
operating_cf = self._get_account_value(
    df, "CF", ["영업활동현금흐름", "영업활동으로인한현금흐름"]
)
capex = abs(self._get_account_value(
    df, "CF", ["유형자산의취득", "유형자산취득",
               "투자활동으로인한유형자산취득"]
))
if operating_cf != 0:
    metrics["free_cash_flow"] = operating_cf - capex
else:
    metrics["free_cash_flow"] = 0
```

### `_get_account_value` (dart_api.py:548-589)

1. 1단계: `account_nm == name` 정확 일치
2. 2단계: `str.contains(name, regex=False)` 부분 일치
3. 둘 다 실패 시 `return 0`

**문제점**:
- 후보 리스트에 공백 없는 형태(`"영업활동현금흐름"`)만 있음
- raw가 `"영업활동으로 인한 현금흐름"`(공백 2개)이면:
  - 1단계 실패 (공백 차이)
  - 2단계도 실패 (`"영업활동현금흐름"`이 substring으로 없음 — 공백 사이에 "으로 인한"이 끼어 있음)
- 결과: `return 0` (silent fail)
- **CAPEX의 후보 `"유형자산의취득"`도 공백 없음. raw는 100% `"유형자산의 취득"` (공백 1개) → 모든 종목에서 매칭 실패**

### CAPEX silent fail의 결과

정상 91개 종목의 FCF 값은 실제로 `OCF − 0 = OCF`이며, 진짜 CAPEX(상당히 큰 값)가 차감되지 않은 과대평가.

표본 검증:
- 효성(2024): raw OCF 4,210억 / raw CAPEX 1,589억 → 진짜 FCF ≈ 2,620억
- 코드 시뮬: OCF=0, CAPEX=0, FCF=0 → 100% 결손
- 삼성전자: raw OCF 85.3조 / raw CAPEX 47.5조 → 진짜 FCF ≈ 37.8조
- 코드 시뮬: OCF=85.3조, CAPEX=0, FCF=85.3조 → **2.25× 과대평가**

## 5. 권고 (수정 방향)

### A) account_id 우선 매칭 (즉시, 핵심)

`_get_account_value`에 `account_id` 후보 인자 추가, 1단계로 IFRS 코드 정확 일치 시도:
```python
def _get_account_value(self, df, sj_div, names, ids=None):
    # 0단계 (신설): account_id 정확 일치 (IFRS 표준 코드)
    # 1단계: account_nm 정확 일치
    # 2단계: account_nm 공백 제거 후 정확 일치 (신설)
    # 3단계: account_nm substring 일치
```

호출부:
```python
operating_cf = self._get_account_value(
    df, "CF",
    names=["영업활동현금흐름", "영업활동으로인한현금흐름"],
    ids=["ifrs-full_CashFlowsFromUsedInOperatingActivities"],
)
```

기대 효과:
- OCF 결손 → 거의 0% (8/8 표본 통과 확인)
- CAPEX는 추가 작업 필요 — K-IFRS 코드 표 작성 또는 공백 정규화

### B) account_nm 공백 정규화 (즉시, 보조)

`name.replace(" ", "")` 와 `account_nm.str.replace(" ", "")` 양쪽 정규화 후 비교 → 5개 변형 중 4개 즉시 잡힘. (Ⅰ. 접두사는 별도)

### C) FCF=0 vs NULL 구분 (선택)

수집 결손과 진짜 0/음수를 분리하기 위해 `free_cash_flow` 컬럼 NULL 허용. 다만 financial_metrics 마이그레이션 + scorer 가드 변경 동반 필요. **별도 PR로 분리 권장**.

### D) sector별 분기 (불필요)

가설 E 기각. 분산 결손이라 sector 분기는 적절하지 않음.
